    • 'use strict' là chế độ nghiêm ngặt, giúp cho chúng ta code JS một cách an toàn hơn. Đây là tính năng mới có từ bản ES6
    • Lợi thế và bất lợi của strict mode:
      • Lợi thế: Với việc phải tuân theo các quy định của strict mode thì code JS sẽ trở nên clean hơn, dễ đọc hơn, ít vướng phải những lỗi không mong muốn.
      • Bất lợi: Khi sử dụng strict mode, JS không còn tính đơn giản, mềm dẻo như khi sử dụng ở chế độ code bình thường, bị giới hạn bởi những quy tắc cứng nhắc.
  • == chỉ so sánh nội dung của biến còn === so sánh cả nội dung và kiểu dữ liệu của 2 biến với nhau.
  • Falsy values: undefined, null, false, 0, -0, 0n, NaN, ""
